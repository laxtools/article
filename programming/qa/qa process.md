# 개요 

 QA (Quality Assurance. 품질 보증)는 전사적인 품질 관리 프로세스를 포함해야 합니다. 

 게임 개발과 서비스에서 QA는 주로 릴리스된 제품의 결함을 발견하고 수정하는 
 과정에만 집중되는 경향이 강하게 있습니다. 이는 개발자들의 자존심과 자율적인 판단에 
 주로 의존하는 게임 개발의 독특한 경향 때문에 비롯되었습니다. 

 개발자들의 역량을 신뢰하고 결함 위주로 점검하여 릴리스하고 서비스하는 방식은 
 벤처 형태로 개발이 진행되던 2000년대까지는 유용했습니다. 특히, 빠르게 개발하고 
 서비스 하는데 많은 기여를 한 점은 분명히 있습니다. 

 하지만, 개별 게임의 기능과 애셋, 기획 데이터를 포함한 컨텐츠가 매우 방대해지고 
 해킹 등 서비스 장애를 일으키는 요소가 예전과 비교할 수 없을 정도로 발전한 
 현재 여전히 유용한 방식이라고 보기 어렵습니다. 

 MMORPG나 MORPG와 같이 다양한 요소를 포함하고 대규모 사용자를 수용해야 하며 
 다양한 공격을 방어해야 하는 게임의 경우 프로세스를 갖춘 QA 체계가 전사적으로 
 확립되어야 안정적인 서비스가 가능하다는 점이 여러 게임의 개발과 서비스를 통해 
 명확해졌다고 할 수 있습니다. 

## 개선할 내용들 

 주로 영역별, 단계별 리뷰가 협업과 품질 보증 과정의 핵심 기능으로 추가되어야 합니다. 

 공식, 비공식적으로 진행되고 있는 현재의 실행 과정을 공식화하고 
 배울만한 점들을 정리해서 프로세스 화 하고 전체에 확산할 필요가 있습니다. 

 리뷰는 공유의 과정이자 개선이 가능한 중요한 사람들간의 협업 지점입니다. 


## 프로세스 개선 지점들 

 품질에 대해 모든 개발자, 사업 담당자, 서비스 주체가 확고한 인식을 가져야 합니다. 
 이런 인식은 인식 자체 보다는 프로세스로 만들어져야 합니다. 

 프로그래밍 단에서는 설계, 코딩, 테스트 과정이 전문적인 QA를 통한 결함 발견 과정 이전에 
 확립되어야 합니다. 소프트웨어 공학과 애자일 기법의 발전 과정에서 핵심으로 얘기되는 
 부분은 다음과 같습니다. 

 - 설계와 설계 리뷰 
 - 코딩과 코드 리뷰 
 - 단위 테스트 
 - 기능 테스트 

 대규모 사용자를 포함하는 경우는 부하 테스트가 필요합니다. 
 기능 테스트의 확장으로 서버의 안정성과 응답 품질을 측정하기 위한 필수적인 방법입니다. 

 기획 단에서도 사용자가 경험하게 될 플레이 경험 (Play Experience. PX) 관점에서 
 기획팀 내외부의 리뷰를 통해 검증해야 합니다. 플레이 테스트가 QA 과정에 
 프로세스로 확립되어 있지 않은 경우가 많기 때문입니다. 기획 단의 전문가들과 함께 
 프로세스를 확립할 필요가 있습니다. 

 그래도 제안 가능한 기본 프로세스는 다음과 같습니다. 

 - 컨셉 기획 리뷰 
 - 시스템 기획 리뷰 
 - 구현 후 플레이 테스트를 통한 개선 
 - 밸런스 리뷰 

 서비스 측면에서 사용자들의 행위를 분석하고 서비스 양태에 대해 정량 분석이 가능하도록 해야 합니다. 
 서비스, 플레이 경험의 중요 특성의 파라미터를 정해서 이를 기반으로 
 품질 기준을 만들고 공유하고 평가하여 개선될 수 있도록 해야 합니다. 

 현재 사용하고 있는 사업적인 파라미터인 CU / AU / NU / DAU / MAU, ARPU / ARPPU 에 더하여 
 사용자별 플레이 타임, 전체 플레이 타임, 게임별 / 컨텐츠별 플레이 타임과 
 게임별 중요 지표들을 찾아서 추가해야 합니다. 

 공통적인 부분과 개별적인 부분을 통합해서 관리할 수 있는 체계와 
 사업과 개발을 통해 사용자 관점에서 개선해 나갈 수 있는 프로세스 구축이 필요합니다. 
 


## 단계적 적용과 지속적인 개선 

 한번에 모든 것을 하기는 어려우므로 단계적으로 찾고 핵심적인 내용들을 적용하고 
 전체 스튜디오, 사업 단에 공유되고 확산되어야 합니다. 

 프로그래밍의 경우 코드 리뷰부터, 기획의 경우 컨셉 리뷰와 플레이 테스트 부터 
 진행할 수 있을 듯 합니다. 

 서비스는 기존 지표들에 추가하여 플레이 타임이나 게임별로 중요한 지표를 찾는 작업이 
 우선되어야 합니다. 전문가들과 논의하여 정리할 필요가 있습니다.
